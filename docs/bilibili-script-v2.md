## 录制指导

### 录制形式：纯录屏 + 旁白

**不建议真人出镜**，原因：
1. 工具类内容，用户关心的是"怎么用"，不是"谁在讲"
2. 录屏演示比真人出镜更直观
3. 降低制作门槛，一个人就能完成

### 录制内容清单

| 时间段 | 录什么 | 怎么录 | 画面要求 |
|--------|--------|--------|----------|
| 0:00-0:05 | 搬运博主的成功案例 | 录屏 B 站搜索"YouTube 搬运" | 快速滚动，不要停顿 |
| 0:05-0:10 | 搬运博主的收益 | 录屏展示高播放量视频 | 用鼠标圈出关键数据 |
| 0:10-0:15 | 播客视频的时长 | 录屏展示播客视频（1-2小时） | 放大显示视频时长 |
| 0:15-0:20 | 播客翻译的痛点 | 录屏展示机翻字幕质量差 | 放大显示糟糕的翻译 |
| 0:20-0:25 | 引出你的工具 | 录屏打开 GitHub 页面 | 鼠标悬停"免费开源" |
| 0:25-0:35 | 工具主界面展示 | 录屏打开工具 | 不要操作，让用户看清楚 |
| 0:35-0:45 | 粘贴链接 + 分析 | 录屏粘贴链接，点击分析 | 放大显示分析结果 |
| 0:45-0:55 | 翻译过程 | 录屏点击生成字幕 | 展示进度条 |
| 0:55-1:05 | 翻译结果展示 | 录屏展示翻译后的字幕 | 放大显示中文字幕 |
| 1:05-1:15 | 字幕编辑功能 | 录屏编辑一条字幕 | 放大显示编辑操作 |
| 1:15-1:25 | 批量处理演示 | 录屏粘贴多个链接 | 展示批量处理进度 |
| 1:25-1:30 | 视频导出 | 录屏点击导出 | 放大显示导出按钮 |
| 1:30-1:40 | 对比表格 | 录屏展示对比表格 | 不要操作，让用户看清楚 |
| 1:40-1:50 | GitHub 下载页面 | 录屏打开 Release 页面 | 放大显示下载按钮 |
| 1:50-1:55 | 结尾 CTA | 录屏停留在 GitHub 页面 | 不要操作 |

### 旁白录制

**录制方式**: 手机录音，不要用专业麦克风（太正式，不亲切）

**语速**: 中等偏快，像跟朋友聊天

**语气**: 轻松、自信、不刻意

**录完后**: 用剪映或 PR 加上字幕，与录屏画面同步
