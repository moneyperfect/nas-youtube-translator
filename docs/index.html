YTSubViewer

面向 30-120 分钟英文演讲/访谈视频的本地优先字幕翻译工作台，从 YouTube 链接一键生成双语字幕与硬字幕视频。

Features

功能介绍

核心能力一览：从输入链接到交付成品，全流程自动化。

智能字幕获取

三级回退策略：优先人工英文字幕 → YouTube 自动字幕 → Whisper 本地转写。自动选择最优字幕源。

DeepSeek 翻译

批量并行翻译，支持术语表、保护词、风格预设。自动检测残留英文并重译低质量句。

多格式导出

输出中文 SRT、中文 ASS、双语 ASS，以及硬字幕 MP4。支持预览版快速导出。

后台任务队列

支持批量入队、3 并发任务、取消/重试。任务状态持久化，刷新页面或重启应用后自动恢复。

字幕编辑器

逐句编辑、单句重译、锁定、批量术语替换。支持问题句筛选和搜索过滤。

质量报告

自动检测空字幕、残留英文、长行、时间重叠、术语一致性，生成 Markdown 格式的质量报告。

创作者配置

按频道/创作者记忆术语表、风格预设和保护词，后续处理同一创作者的视频时自动继承。

多语言界面

中英文双语界面切换，支持中文、日语、韩语作为翻译目标语言。语言偏好持久化。

How It Works

工作原理

从 YouTube 链接到成品视频，经过 7 个自动化阶段。

1. 视频分析

通过 yt-dlp 提取视频元数据：标题、时长、缩略图、可用字幕轨道。自动判断最优字幕来源策略。

YouTubeService.extract_metadata()

2. 视频下载

下载最高 1080p MP4 格式视频到本地工作目录。支持断点续传和已有文件复用。

YouTubeService.download_video()

3. 字幕获取

按优先级获取英文字幕：人工上传字幕 > YouTube 自动字幕 > Whisper 本地转写。Whisper 首次使用需下载模型。

YouTubeService / TranscriptionService

4. 分段翻译

英文字幕按批次发送到 DeepSeek API 翻译。3 并行 worker，每批 20 条，自动重试与质量自检。

DeepSeekTranslator.translate_cues_stream()

5. 后处理与质检

优化中文断句（按行宽限制），检测残留英文自动重译，生成质量报告（Markdown）。

polish_translated_cues() + QualityReport

6. 字幕生成

输出中文 SRT、中文 ASS、双语 ASS 三种格式。ASS 使用微软雅黑字体，支持自定义样式。

write_srt() + write_ass()

7. 视频导出

使用 ffmpeg 将 ASS 字幕烧录进视频，生成硬字幕 MP4。支持预览版（快速低质量）和正式版。

VideoExportService.burn_video_events()

Architecture

系统架构

分层架构设计，前端 → API → Pipeline → Service 层层解耦。

前端层
index.html app.js i18n.js styles.css Gradio Legacy UI
↓ HTTP / WebSocket
API 层
webapp.py (FastAPI) auth.py rate_limit.py
↓ 调用

pipeline.py background_jobs.py job_state.py
↓ 委托
服务层

↓ 调用
外部依赖
yt-dlp ffmpeg mpv faster-whisper DeepSeek API
关键设计决策： Pipeline 层（pipeline.py）是整个系统的核心编排器，负责串联各服务层模块，管理翻译进度的持久化与断点恢复。后台任务管理器（background_jobs.py）基于 threading 实现单机并发队列，任务状态以 JSON 文件持久化。

Development

开发过程

从需求分析到产品迭代的完整开发历程。

Sprint 1 — 核心链路

MVP：跑通 YouTube → 翻译 → 字幕 全流程

搭建 FastAPI 后端 + Gradio 前端原型，接入 yt-dlp 下载、DeepSeek 翻译、SRT 导出。

  • 初始化 Python 项目结构，配置虚拟环境
  • 实现 YouTubeService（yt-dlp 封装）
  • 实现 TranscriptionService（faster-whisper 本地转写）
  • 实现 DeepSeekTranslator（批量翻译 + 术语表）
  • 实现 SubtitlePipeline 编排全链路
  • 导出中文/双语 SRT 字幕文件

Sprint 2 — 任务系统与编辑器

后台任务队列、字幕编辑器、质量报告

引入 BackgroundGenerationManager 实现后台并发，添加字幕逐句编辑和质量检测。

  • 实现 BackgroundGenerationManager（threading 并发队列）
  • 任务状态 JSON 持久化，支持崩溃恢复
  • 字幕编辑器：逐句编辑、重译、锁定、批量替换
  • 质量报告系统：空字幕、残留英文、长行检测
  • ASS 字幕格式支持（中文 + 双语模式）
  • 硬字幕视频导出（ffmpeg 字幕烧录）

Sprint 3 — 现代 Web UI

从 Gradio 迁移到自研前端，创作者配置

构建 Vanilla JS 现代 Web UI，替代 Gradio 作为主界面。添加频道级配置记忆。

  • 设计并实现 index.html + app.js + styles.css
  • 卡片式布局，响应式设计，实时进度反馈
  • 实现 CreatorProfileStore（按频道记忆配置）
  • 三种性能模式（fast / balanced / quality）
  • 批量链接入队、预览版导出
  • License 激活系统

Sprint 4 — 产品加固

安全性、性能、可部署性、国际化全面升级

针对上线网站定位，对安全性、性能、跨平台、i18n 四个维度进行系统性升级。

  • 安全：API Token 认证、API Key 加密存储、CORS、速率限制、URL 校验
  • 性能：3 并发任务、翻译重试上限、ffmpeg/Whisper 超时、磁盘检查
  • 部署：跨平台兼容、Dockerfile、docker-compose、GitHub Actions CI/CD
  • 国际化：中英文双语界面、i18n 框架、多目标语言翻译（CN/JA/KR）
